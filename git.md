# Git Command Line Operations 

## Introduction
This technical paper is on  Git command-line operations and fundamental version control concepts. Git is widely used by software developers to track changes, collaborate efficiently, and manage code history.

## Initializing a Git Repository

A Git repository must be initialized  to sart a project .
 * git init

## Adding a File
* touch a.txt

## Making a Commit
* git commit -m "Initial commit"

## Creating a Branch
* git branch leaf

## Adding Another File
* git add b.txt

## Creating a Second Commit
* git commit -m "Added b.txt"

## Merging Branches
Switch back to main branch
* git checkout master
Merge the branch
* git merge "branchname"

## What is staging area?
* Files are added using git add
* Only staged changes are committed

## Where is HEAD Right Now?
Head shows your current commit you are working on
* git branch
